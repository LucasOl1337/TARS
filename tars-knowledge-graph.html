Mapa dos Principais Fluxos

1. BOOT
start-tars.ps1 → backend (lifespan: init_db + seed persona + poll) → dashboard (Vite + proxy)
2. CHAT
POST /chat → build_system_prompt (persona DB + tools) → provider_for_model → LLM
3. TOOL
LLM pede → /tools/{id}/invoke → builtin (Python) OU bridge proxy + log_echo obrigatório
4. PROXY
Qualquer chamada → call_bridge (httpx) → log_echo (trace_id) → envelope canônico
5. OBSERVABILIDADE
Echoes/flows + bridges/status + ports + services (derivado)
Pontos Críticos / Riscos
  • • Zero autenticação em todo o sistema
  • • Persona dual confusa (local SQLite vs /persona/active do Yume)
  • • Chat (principal feature) sem nenhuma UI
  • • Forte dependência de Yume + Kamui externos
  • • Duplicação massiva (CORVO + frontend + backend/src)
  • • Sem migrations no SQLite
